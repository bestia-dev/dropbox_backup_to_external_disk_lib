5//! # decrypt dropbox api token from file or use the oauth2 "PKCE code workflow" to get the access token and encrypt it and save into file
6//!
7//! Dropbox offers the preferred OAuth2 "PKCE with code flow" for client applications.
8//! In clients that run on the client machine it is impossible to hide secrets.
9//! So there is the new recommended PKCE "pixie" (Proof Key for Code Exchange) to the rescue.
10//! <https://dropbox.tech/developers/pkce--what-and-why->
11//!
12//! ## Secrets
13//!
14//! In this module there will be a lot of work with secrets.  
15//! It is difficult to trust an external crate with your secrets.  
16//! External crates can get updated unexpectedly and change to malicious code.  
17//!
18//! ## Copy code instead of dependency crate
19//!
20//! It is best to have the Rust code under your fingertips when dealing with secrets.  
21//! Than you know, nobody will touch this code except of you.  
22//! You can copy this code directly into your codebase as a module,
23//! inspect and review it and know exactly what is going on.  
24//! The code is as linear and readable with comments as possible.
25//!
26//! ## Store encrypted secret to file
27//!
28//! The secrets will be encrypted with an ssh private key and stored in the `~/.ssh` folder.  
29//! This way the data is protected at rest in storage drive.  
30//!
31//! ## In memory protection
32//!
33//! This is a tough one! There is no 100% software protection of secrets in memory.  
34//! Theoretically an attacker could dump the memory in any moment and read the secrets.  
35//! There is always a moment when the secret is used in its plaintext form. This cannot be avoided.
36//! All we can do now is to be alert what data is secret and take better care of it.  
37//! Every variable that have secrets will have the word `secret` in it.
38//! When a variable is confusing I will use the word `plain` to express it is `not a secret`.
39//! To avoid leaking in logs I will use the `secrecy` crate. This is not 100% protection. It is important just to express intent when the secrets are really used.  
40//! `Secrecy` needs the trait `zeroize` to empty the memory after use for better memory hygiene.
41//! I will add the type names explicitly to emphasis the secrecy types used.
42//!
43//! ## encrypt_decrypt_with_ssh_key_mod
44//!
45//! This module depends on the generic module for encryption `encrypt_decrypt_with_ssh_key_mod.rs`. That module also needs to be copy and paste into your project.
46//!
47//! ## Other dependencies
48//!
49//! In `Cargo.toml` there are a group od dependencies needed for this to work. They are so generic that I don't expect any malware in them to be able to steal some usable secrets.  
50//!
51//! Beware that the versions of crates in `Cargo.toml` are not precisely pinpointed. In rust the symbol '=' means "the same major number equal or newer to". This means from one compilation to another, it can automatically change to a newer version without the programmer even noticing it.
52//!
53//! This is great if the newer version is solving some security issue. But this is super-bad if the newer version is malware supply chain attack. We have no idea how to distinguish one from another.
54//!
55//! Just to mention: there exists the trick to control the `Cargo.lock` file and forbid the change of the version number, but more times than not, you will not want to commit the lock file into the Dropbox repository.
56//!
57//! ```toml
58//! [dependencies]
59//! reqwest={version="0.12.12", features=["json","blocking"]}
60//! serde ={ version= "1.0.217", features=["std","derive"]}
61//! serde_json = "1.0.138"
62//! ssh-key = { version = "0.6.7", features = [ "rsa", "encryption","ed25519"] }
63//! ssh_agent_client_rs_git_bash = "0.0.19"
64//! rsa = { version = "0.9.7", features = ["sha2","pem"] }
65//! zeroize = {version="1.8.1", features=["derive"]}
66//! aes-gcm = "0.10.3"
67//! base64ct = {version = "1.6.0", features = ["alloc"] }
68//! secrecy = "0.10.3"
69//! chrono ="0.4.39"
70//! crossplatform_path="1.1.1"
71//! ```
72//!

111#[derive(serde::Deserialize, serde::Serialize, zeroize::Zeroize, zeroize::ZeroizeOnDrop, Debug)]
112struct SecretResponseAccessToken {
113    access_token: String,
114    expires_in: i64,
115    refresh_token: Option<String>,
116}

118/// Application state (static) is initialized only once in the main() function.
119///
120/// And then is accessible all over the code.
121pub fn dropbox_api_config_initialize() {
122    if DROPBOX_API_CONFIG.get().is_some() {
123        return;
124    }
125
126    let dropbox_api_config_json = std::fs::read_to_string("dropbox_api_config.json")
127        .unwrap_or_else(|_| panic!("{RED}Error: The file dropbox_api_config.json is missing.{RESET}"));
128    let dropbox_api_config: DropboxApiConfig = serde_json::from_str(&dropbox_api_config_json)
129        .unwrap_or_else(|_| panic!("{RED}Error: The content of dropbox_api_config.json is not correct.{RESET}"));
130    let _ = DROPBOX_API_CONFIG.set(dropbox_api_config);
131}

133/// Start the dropbox oauth2 PKCE code workflow
134/// It will use the private key from the .ssh folder.
135/// The encrypted file has the same file name with the ".enc" extension.
136/// Returns access_token to use as bearer for api calls
137pub fn get_dropbox_secret_token(client_id: &str) -> Result<SecretString> {
138    let private_key_file_name = DROPBOX_API_CONFIG
139        .get()
140        .ok_or_else(|| Error::ErrorFromStr("DROPBOX_API_CONFIG.get error"))?
141        .dropbox_api_private_key_file_name
142        .to_string();
143    println!("  {YELLOW}Check if the ssh private key exists.{RESET}");
144    let private_key_path_struct = ende::PathStructInSshFolder::new(private_key_file_name.clone())?;
145    if !private_key_path_struct.exists() {
146        println!("{RED}Error: Private key {private_key_path_struct} does not exist.{RESET}");
147        println!("  {YELLOW}Create the private key in bash terminal:{RESET}");
148        println!(r#"{GREEN}ssh-keygen -t ed25519 -f {private_key_path_struct} -C "dropbox api secret_token"{RESET}"#);
149        return Err(Error::ErrorFromStr("Private key file not found."));
150    }
151
152    println!("  {YELLOW}Check if the encrypted file exists.{RESET}");
153    let encrypted_path_struct = ende::PathStructInSshFolder::new(format!("{private_key_file_name}.enc"))?;
154    if !encrypted_path_struct.exists() {
155        println!("  {YELLOW}Encrypted file {encrypted_path_struct} does not exist.{RESET}");
156        println!("  {YELLOW}Continue to authentication with the browser{RESET}");
157        let secret_access_token = authenticate_with_browser_and_save_file(client_id, &private_key_path_struct, &encrypted_path_struct)?;
158        Ok(secret_access_token)
159    } else {
160        println!("  {YELLOW}Encrypted file {encrypted_path_struct} exist.{RESET}");
161        let plain_file_text = ende::open_file_b64_get_string(encrypted_path_struct.get_cross_path())?;
162        // deserialize json into struct
163        let encrypted_text_with_metadata: ende::EncryptedTextWithMetadata = serde_json::from_str(&plain_file_text)?;
164
165        // check the expiration
166        let utc_now = chrono::Utc::now();
167
168        if encrypted_text_with_metadata.access_token_expiration.is_none() {
169            return Err(Error::ErrorFromStr("access_token_expiration is None"));
170        }
171
172        let access_token_expiration = chrono::DateTime::parse_from_rfc3339(
173            encrypted_text_with_metadata
174                .access_token_expiration
175                .as_ref()
176                .expect("The former line asserts this is never None"),
177        )?;
178        if access_token_expiration <= utc_now {
179            println!("{RED}Access token has expired, use refresh token{RESET}");
180            let secret_decrypted_from_file = decrypt_text_with_metadata(encrypted_text_with_metadata)?;
181            let secret_response_access_token: SecretBox<SecretResponseAccessToken> = refresh_tokens(
182                client_id,
183                secret_decrypted_from_file
184                    .expose_secret()
185                    .refresh_token
186                    .clone()
187                    .ok_or_else(|| Error::ErrorFromStr("refresh_token is None"))?,
188            )?;
189
190            let secret_access_token = SecretString::from(secret_response_access_token.expose_secret().access_token.to_string());
191            let expires_in = secret_response_access_token.expose_secret().expires_in;
192
193            // create new secret_response_access_token, because the response of refresh does not have the refresh_token
194            let secret_response_access_token = SecretResponseAccessToken {
195                access_token: secret_access_token.expose_secret().to_string(),
196                expires_in,
197                refresh_token: secret_decrypted_from_file.expose_secret().refresh_token.clone(),
198            };
199            let secret_response_access_token = SecretBox::new(Box::new(secret_response_access_token));
200
201            println!("  {YELLOW}Encrypt data and save file{RESET}");
202            encrypt_and_save_file(&private_key_path_struct, &encrypted_path_struct, secret_response_access_token)?;
203            return Ok(secret_access_token);
204        }
205        println!("  {YELLOW}Decrypt the file with the private key.{RESET}");
206        let secret_response_access_token = decrypt_text_with_metadata(encrypted_text_with_metadata)?;
207        let secret_access_token = SecretString::from(secret_response_access_token.expose_secret().access_token.clone());
208        Ok(secret_access_token)
209    }
210}

212fn authenticate_with_browser_and_save_file(
213    client_id: &str,
214    private_key_path_struct: &ende::PathStructInSshFolder,
215    encrypted_path_struct: &ende::PathStructInSshFolder,
216) -> Result<SecretString> {
217    let secret_response_access_token: SecretBox<SecretResponseAccessToken> = authentication_with_browser(client_id)?;
218    let secret_access_token = SecretString::from(secret_response_access_token.expose_secret().access_token.clone());
219    println!("  {YELLOW}Encrypt data and save file{RESET}");
220
221    encrypt_and_save_file(private_key_path_struct, encrypted_path_struct, secret_response_access_token)?;
222    Ok(secret_access_token)
223}

225/// Oauth2 PKCE code flow needs to be authenticated with a browser
226/// <https://dropbox.tech/developers/pkce--what-and-why->
227fn authentication_with_browser(client_id: &str) -> Result<SecretBox<SecretResponseAccessToken>> {
228    // Step 1: Client creates code_verifier and subsequent code_challenge
229    // generate a random 32 bytes, then encode to base64
230
231    let mut random_bytes = [0_u8; 32];
232    use aes_gcm::aead::rand_core::RngCore;
233    aes_gcm::aead::OsRng.fill_bytes(&mut random_bytes);
234    let code_verifier = <base64ct::Base64UrlUnpadded as base64ct::Encoding>::encode_string(&random_bytes);
235
236    let code_challenge_bytes = rsa::sha2::Sha256::digest(code_verifier.as_bytes());
237    let code_challenge = <base64ct::Base64UrlUnpadded as base64ct::Encoding>::encode_string(&code_challenge_bytes);
238
239    // Step 2: Client sends code_challenge and code_challenge_method to /oauth2/authorize
240    let url = format!("https://www.dropbox.com/oauth2/authorize?client_id={client_id}&response_type=code&token_access_type=offline&code_challenge={code_challenge}&code_challenge_method=S256");
241    println!("  {YELLOW}Open authorization web page in browser and allow access: {RESET}");
242    println!("{GREEN}{url}{RESET}");
243
244    println!("  {YELLOW}Copy-paste the access_code:{RESET}");
245
246    // the access code could be sent to to the redirect_uri. But I don't know if that makes it easier for me.
247    // Instead the user can just copy paste from the browser. I think it needs to be done just once.
248    let access_code = inquire::Password::new("").without_confirmation().prompt()?;
249
250    // Step 4: Client sends authorization_code and code_verifier to /oauth2/token
251    let params = [
252        ("grant_type", "authorization_code"),
253        ("code", &access_code),
254        ("code_verifier", &code_verifier),
255        ("client_id", client_id),
256    ];
257
258    let request = reqwest::blocking::Client::new()
259        .post("https://api.dropbox.com/oauth2/token")
260        .form(&params);
261
262    let response = request.send()?;
263    let response_str = response.text()?;
264
265    /*
266    response json
267    {
268    "access_token": "NW7lYmEWHgUAAAAAAAAAAbeutI8iL5CuBik9_CPD5r83XvcQPt-7O5diOdUUcsuX",
269    "expires_in": 14399,
270    "refresh_token": "xxxx",
271    }
272    */
273
274    let secret_response_access_token: SecretResponseAccessToken = serde_json::from_str(&response_str)?;
275
276    Ok(SecretBox::new(Box::new(secret_response_access_token)))
277}

279/// use refresh token to get new access_token and refresh_token
280fn refresh_tokens(client_id: &str, refresh_token: String) -> Result<SecretBox<SecretResponseAccessToken>> {
281    // https://developers.dropbox.com/oauth-guide#implementing-oauth
282
283    #[derive(serde::Serialize)]
284    struct RequestWithRefreshToken {
285        client_id: String,
286        grant_type: String,
287        refresh_token: String,
288    }
289    // TODO: refresh for dropbox
290    println!("  {YELLOW}Send request with refresh_token and retrieve access tokens{RESET}");
291    println!("  {YELLOW}wait...{RESET}");
292
293    let params = [
294        ("grant_type", "refresh_token"),
295        ("refresh_token", &refresh_token),
296        ("client_id", client_id),
297    ];
298
299    let request = reqwest::blocking::Client::new()
300        .post("https://api.dropbox.com/oauth2/token")
301        .form(&params);
302
303    let response = request.send()?;
304    let response_str = response.text()?;
305    let secret_response_refresh_token: SecretResponseAccessToken = serde_json::from_str(&response_str)?;
306
307    Ok(SecretBox::new(Box::new(secret_response_refresh_token)))
308}

310/// encrypt and save file
311///
312/// The "seed" are just some random 32 bytes.
313/// The "seed" will be "signed" with the private key.
314/// Only the "owner" can unlock the private key and sign correctly.
315/// This signature will be used as the true passcode for symmetrical encryption.
316/// The "seed" and the private key path will be stored in plain text in the file
317/// together with the encrypted data in json format.
318/// To avoid plain text in the end encode in base64 just for obfuscate a little bit.
319fn encrypt_and_save_file(
320    private_key_path_struct: &ende::PathStructInSshFolder,
321    encrypted_path_struct: &ende::PathStructInSshFolder,
322    secret_response_access_token: SecretBox<SecretResponseAccessToken>,
323) -> Result<()> {
324    let secret_string = SecretString::from(serde_json::to_string(&secret_response_access_token.expose_secret())?);
325
326    let (plain_seed_bytes_32bytes, plain_seed_string) = crate::encrypt_decrypt_mod::random_seed_32bytes_and_string()?;
327
328    println!("  {YELLOW}Unlock private key to encrypt the secret symmetrically{RESET}");
329    let secret_passcode_32bytes: SecretBox<[u8; 32]> =
330        ende::sign_seed_with_ssh_agent_or_private_key_file(private_key_path_struct, plain_seed_bytes_32bytes)?;
331
332    println!("  {YELLOW}Encrypt the secret symmetrically {RESET}");
333    let encrypted_string = ende::encrypt_symmetric(secret_passcode_32bytes, secret_string)?;
334
335    // the file will contain json with 3 plain text fields: fingerprint, seed, encrypted, expiration
336
337    // calculate expiration minus 10 minutes or 600 seconds
338    let utc_now = chrono::Utc::now();
339    let access_token_expiration = utc_now
340        .checked_add_signed(chrono::Duration::seconds(
341            secret_response_access_token.expose_secret().expires_in - 600,
342        ))
343        .ok_or_else(|| Error::ErrorFromStr("checked_add_signed"))?
344        .to_rfc3339();
345
346    let encrypted_text_with_metadata = ende::EncryptedTextWithMetadata {
347        private_key_file_name: private_key_path_struct.get_file_name().to_string(),
348        plain_seed_string,
349        plain_encrypted_text: encrypted_string,
350        access_token_expiration: Some(access_token_expiration),
351        token_name: None,
352    };
353    let plain_file_text = serde_json::to_string_pretty(&encrypted_text_with_metadata)?;
354    // encode it just to obscure it a little bit
355    let file_text = ende::encode64_from_string_to_string(&plain_file_text);
356
357    let mut file = std::fs::File::create(encrypted_path_struct.get_full_file_path())?;
358    #[cfg(target_family = "unix")]
359    {
360        let metadata = file.metadata()?;
361        let mut permissions = metadata.permissions();
362        std::os::unix::fs::PermissionsExt::set_mode(&mut permissions, 0o600);
363    }
364    std::io::Write::write_all(&mut file, file_text.as_bytes())?;
365
366    println!("  {YELLOW}Encrypted text saved to file.{RESET}");
367
368    Ok(())
369}

371/// decrypt text with metadata
372///
373/// The encrypted file is encoded in base64 just to obfuscate it a little bit.  
374/// In json format in plain text there is the "seed", the private key path and the encrypted secret.  
375/// The "seed" will be "signed" with the private key.  
376/// Only the "owner" can unlock the private key and sign correctly.  
377/// This signature will be used as the true passcode for symmetrical decryption.  
378fn decrypt_text_with_metadata(
379    encrypted_text_with_metadata: ende::EncryptedTextWithMetadata,
380) -> Result<SecretBox<SecretResponseAccessToken>> {
381    // the private key file is written inside the file
382    let private_key_path_struct = ende::PathStructInSshFolder::new(encrypted_text_with_metadata.private_key_file_name.clone())?;
383    if !private_key_path_struct.exists() {
384        return Err(Error::ErrorFromStr(
385            "{RED}Error: File {private_key_path_struct} does not exist! {RESET}",
386        ));
387    }
388
389    let plain_seed_bytes_32bytes = ende::decode64_from_string_to_32bytes(&encrypted_text_with_metadata.plain_seed_string)?;
390    // first try to use the private key from ssh-agent, else use the private file with user interaction
391    let secret_passcode_32bytes: SecretBox<[u8; 32]> =
392        ende::sign_seed_with_ssh_agent_or_private_key_file(&private_key_path_struct, plain_seed_bytes_32bytes)?;
393    // decrypt the data
394    let decrypted_string = ende::decrypt_symmetric(secret_passcode_32bytes, encrypted_text_with_metadata.plain_encrypted_text)?;
395    // parse json to struct
396    let secret_response_access_token: SecretBox<SecretResponseAccessToken> =
397        SecretBox::new(Box::new(serde_json::from_str(decrypted_string.expose_secret())?));
398    Ok(secret_response_access_token)
